# Iris Species Predictor - Flask ML App (No Docker)

Project 1 of 2 for this assignment. A lightweight, accurate ML web app that predicts the species of an iris flower (setosa, versicolor, virginica) from four measurements. Deployed on Render without Docker, using Render's native Python runtime.

The Docker-based counterpart to this assignment (a separate ML task - diabetes progression prediction) lives in its own repo: diabetes-risk-predictor-docker.

## Dataset

Classic Iris dataset (Fisher, 1936) - the same 150-row dataset distributed on the UCI Machine Learning Repository and as "Iris Species" on Kaggle: https://www.kaggle.com/datasets/uciml/iris

Loaded here via sklearn.datasets.load_iris (identical values, no download needed, keeps the repo self-contained and reproducible).

Chosen deliberately: 3 perfectly balanced classes, no missing values, and small enough that the trained model stays under 100KB while still reaching 95-100% test accuracy - a genuinely light and accurate deployable model.

## Made for a general audience

Beyond the raw prediction, the web form includes:

- An inline diagram explaining what "sepal" and "petal" actually mean, with length/width labeled
- Typical value ranges shown under each input field, so users know what reasonable numbers look like
- A short description and reference photo of the predicted species after each prediction

## Project structure

- train.py - EDA, feature engineering, GridSearchCV, saves iris_model.pkl
- app.py - Flask app: web form ("/") and JSON API ("/api/predict")
- templates/index.html - Web UI (diagram, range hints, species info)
- iris_model.pkl - Trained scikit-learn pipeline (StandardScaler + RandomForest)
- requirements.txt - Python dependencies
- .gitignore
- eda_*.png - EDA plots generated by train.py (pairplot, correlation heatmap, confusion matrix)

No Dockerfile here on purpose - this project demonstrates the non-Docker Flask deployment path.

## Run locally

Install dependencies:

    pip install -r requirements.txt

Retrain the model (regenerates iris_model.pkl and EDA plots):

    python train.py

Start the Flask dev server on http://localhost:5000:

    python app.py

## API usage

Example request:

    curl -X POST http://HOST/api/predict -H "Content-Type: application/json" -d "{\"sepal_length\": 5.1, \"sepal_width\": 3.5, \"petal_length\": 1.4, \"petal_width\": 0.2}"

Example response:

    {
      "input": {"sepal_length": 5.1, "sepal_width": 3.5, "petal_length": 1.4, "petal_width": 0.2},
      "prediction": {
        "species": "setosa",
        "confidence": 1.0,
        "probabilities": {"setosa": 1.0, "versicolor": 0.0, "virginica": 0.0},
        "info": {
          "label": "Iris setosa",
          "description": "The smallest of the three species, with short, wide petals and a compact flower...",
          "image": "https://upload.wikimedia.org/wikipedia/commons/5/56/Kosaciec_szczecinkowaty_Iris_setosa.jpg"
        }

## Deployment (Render, no Docker)

Render's native Python runtime is used:

- Build command: pip install -r requirements.txt
- Start command: gunicorn app:app

Live app: https://iris-flower-classifier-4jw0.onrender.com

GitHub repo: https://github.com/Ayush-1271/iris-flower-classifier

## Model details

- Algorithm: Random Forest Classifier (scikit-learn Pipeline with StandardScaler)
- Tuning: GridSearchCV, 5-fold cross-validation, over n_estimators, max_depth, min_samples_leaf
- Features: 4 raw measurements plus 2 engineered features (petal_area, sepal_area)
- Evaluation: accuracy, precision/recall/F1 per class, confusion matrix (see eda_confusion_matrix.png)
